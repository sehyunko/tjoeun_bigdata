5장. 서브 쿼리 (SUB-QUERY) 

서브쿼리: 쿼리문 안에 또 다시 쿼리문이 포함된 형태 -> 쿼리문을 작성할 때에는 반드시 괄호로 둘러 싸서 작성 하여야 한다 


(1) 단일행 서브 쿼리 

쿼리문 결과 하나의 단일행 결과값이 나오는 서브쿼리 -> 단일행 서브 쿼리의 경우 단일행 비교연산자 사용이 가능하다 (ex. = > < !=) 


(2) 다중행 서브쿼리 

쿼리문 결과 다중행 결과 값이 나오는 서브쿼리 -> 다중행 서브쿼리의 경우 단일행 비교연산자의 사용이 불가능 하다.  대신 다중행 비교연산자를 

사용하여야 한다. (ex. IN, ANY, SOME, ALL, EXISTS) 


다중행 비교연산자 

1. IN (서브 쿼리) : OR문과 유사하다고 보면 된다. 서브 쿼리 결과 나오는 값들 중 하나라도 참이면 참 

2. ~ =/>/</!= ANY,SOME (서브쿼리): IN과 비슷하게 서브쿼리의 결과 값들 중 하나 이상 참이면 결과 값이 참이게 된다 

3. EXISTS (서브 쿼리): 서브 쿼리의 결과 값 중 하나라도 참이면 참 

4. ~ =/>/</!= ALL (서브 쿼리): AND 연산자와 유사하게 서브쿼리의 모든 결과가 참이어야 결과값이 참이게 된다
