9장. 인덱스(INDEX)와 트랜잭션(TRANSACTION) 

1. 인덱스 (INDEX) 

자료 검색을 빠르게 하기 위한 도구로서 쓰인다. 

인덱스로 지정된 필드의 데이터들은 B*트리 형식이라는 형태로 자료가 저장되는데, 이를 통해 

빠른 자료 탐색이 가능하다. 

하지만 새 데이터가 추가되거나, 기존 데이터가 삭제될 시에 B*트리 구조가 전부 재배열 되어야 한다는 점에서

비효율적인면도 있기 때문에 너무 인덱스 필드를 많이 남발하지 않는 것이 좋다 

** B*트리 구조
입력한 인덱스 수를 기준으로 다음 입력한 인덱스 수가 크면 왼쪽, 작으면 오른쪽으로 입력을 하며 트리구조를 계속 아래로 

내려가면서 작성한다. 마지막 단계에서 트리를 납작하게 압출하면 자연스럽게 크기 순으로 인덱스 값들이 정렬된다 


CREATE INDEX 인덱스명 
	ON 테이블명 (필드명); 


2. 트랜잭션 (TRANSACTION) 

트랜잭션은 데이터 처리의 한 단위이다. DML문장을 수행하면 수행한 순서대로 트랜잭션에 쌓이게 된다. 

COMMIT; 문장을 수행하면 그동안 쌓여 있었던 수행문들이 데이터 베이스에 적용되게 되고, 

ROLLBACK;을 적용하게 되면 트랜잭션에 쌓여있던 모든 수행문을 취소한다. 

SAVEPOINT; -> 트랜잭션의 구간 구간을 SAVEPOINT로 지정해주고, ROLLBACK을 수행할 때 모든 트랜잭션을 취소하지 않고, 

특정 SAVEPOINT 구간의 이후 ~ 끝 구간만 취소한다
