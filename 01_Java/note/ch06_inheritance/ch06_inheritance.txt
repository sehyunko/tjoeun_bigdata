6장. 상속 (Inheritance) 

1. 상속의 개념: 부모 클래스(상위 클래스, super 클래스)로 부터 맴버 변수, 메소드들을 물려받는 것 
(단, private으로 설정된 맴버 변수나 메소드는 직접 제어할 수 없다)

접근제어자 [final/abstract] class 클래스 이름 extends 상위클래스{
	추가할 맴버 변수 선언;
	생성자; -> 상위 클래스가 매개변수 있는 생성자만 가지고 있는 경우 super(value1,value2...) 를 명시해줘서 
		 그대로 매개변수 있는 생성자를 이어 받아야함 
	추가할 메소드 선언 ;
}

** 자바에서는 여러개의 상위 클래스를 물려 받는 다중 상속은 지원 되지 않는다

2. 추상화 

	     Object class (모든 클래스들의 최상위 클래스) 
		↑
	          A class

	↗		↖
          B class		C class

위 도식과 같이 상속 관계가 있다고 할 때, B와 C클래스는 A라는 공통 부모 클래스 타입으로 정의 될 수 있다.

보통 여러 객체를 생성할 때 추상화 작업을 통해 하나의 공통 클래스 타입으로 배열을 정의 한 뒤 배열로 묶어 한번에 정의하는데 쓰이기도 함. 

3. 오버라이드 (Override)와 오버 로딩 (Overloading)

오버라이드: 부모 클래스로 부터 이어 받은 일반 메소드를 재정의 하여 자신이 원하는 기능을 추가하거나 다른 기능으로 변경하는 것

오버로딩: 같은 이름의 메소드를 하나 더 생성 하것. 예를 들어, 매개 변수가 없는 일반 메소드가 있다면, 똑같은 이름의 메소드를 매개 변수가 있는 
	메소드로 정의하여 추가적인 기능을 하도록 정의
