5장. 패키지 

수많은 자바 파일(클래스)들을 묶어 관리하기 위함. 관련있는 자바 파일들을 담아놓은 폴더와 같은 개념.

패키지의 이름이 중복되지 않게 하기 위해 보통 자사의 도메인주소 역순으로 패키지 이름을 만든다 (ex. com.tjoeun.*) 

다른 패키지에 담긴 클래스를 이용하려 할 경우 import 과정이 필요함. 동일 패키지내 클래스를 사용할 경우 import 과정은 필요없음
-> import com.tjoeun.(패키지명)   
-> import com.tjoeun.* :이렇게 명시 해줄 경우 패키지 내 모든 클래스들을 불러옴. 보통 이렇게 해서 작업 후 ctrl+shift+o를 눌러 사용하지않은 
		      클래스들은 마지막에 누락시켜줌.
